09 · Attack & Defense

보안 — 각 계층의 약점과 방어

네트워크 보안의 기본은 "어디서 누가 무엇을 할 수 있는지"를 정확히 그려보는 것입니다. 프로토콜의 설계 가정이 깨지는 순간 — 인증이 없거나, 신뢰가 잘못 두어졌거나 — 거기서 공격이 발생합니다. 아래는 대표적인 시나리오와 방어를 계층 순으로 정리한 것입니다.

이 페이지의 내용은 교육·방어 목적입니다. 운영 중인 네트워크에 동의 없이 시도하지 마세요.
01 · L2 MITM

ARP 스푸핑

ARP는 응답을 검증하지 않습니다. 공격자 X가 "192.168.0.1(게이트웨이)은 내 MAC이다"라고 피해자 A의 ARP 캐시를 덮어쓰면, A가 외부로 보내는 모든 프레임이 X를 거쳐 갑니다. X가 그대로 게이트웨이로 forwarding해 주면 A는 정상 동작처럼 느끼지만, 도청·변조가 가능해집니다.

방어

  • 스위치에서 DAI (Dynamic ARP Inspection) + DHCP snooping
  • 중요 호스트에 정적 ARP 항목
  • 802.1X로 비인가 단말 자체를 LAN에 못 붙게
  • L7 암호화(TLS): 경로는 빼앗겨도 내용은 못 읽게
02 · DNS

DNS 캐시 포이즈닝

DNS 응답은 UDP·평문 기반입니다. 공격자가 리졸버보다 빨리 가짜 응답을 만들어 TX ID·소스 포트를 맞추면, 리졸버는 그것을 진짜로 믿고 TTL 동안 캐시합니다. 그 캐시를 보는 모든 사용자가 가짜 IP로 안내됩니다.

방어

  • 소스 포트 무작위화(Kaminsky 사건 후 표준화) — 추측 공간을 키움
  • DNSSEC — 응답의 디지털 서명을 검증
  • DoT/DoH — 도청·삽입 공격 자체를 차단
03 · L4

SYN Flood — 자원 고갈

TCP 서버는 SYN을 받으면 "half-open" 상태로 자원을 잡고 SYN+ACK를 보냅니다. 공격자가 출발지 IP를 위조해 수많은 SYN만 던지면, 서버의 백로그 큐가 가득 차서 정상 사용자의 연결을 받지 못합니다.

방어

  • SYN cookie: 백로그를 잡지 않고 응답 안에 상태를 인코딩
  • 방화벽·로드밸런서의 SYN proxy / rate limit
  • uRPF, 소스 IP 검증으로 위조 트래픽 일부 차단
04 · UDP

반사·증폭 DDoS

공격자가 작은 요청에 큰 응답을 돌려주는 서비스(DNS, NTP, memcached)에 피해자의 IP를 출발지로 위조해 요청을 던집니다. 응답은 피해자에게 쏟아져, 공격자는 자기 대역폭의 수십~수만 배를 폭격에 동원할 수 있습니다.

서비스대략적 증폭률이유
DNS (ANY)30~50x큰 레코드(특히 DNSSEC)
NTP (monlist)~500x최근 클라이언트 목록 반환
memcached~50,000x대량 키 응답 (인증 부재)

방어

  • ISP의 BCP38(출발지 위조 차단) — 가장 근본적
  • 증폭 가능한 서비스를 인터넷에 노출하지 않음
  • 응답 크기 제한, 인증 강제
  • 방어 측: CDN/anycast로 트래픽 분산, 스크러빙 센터

방어 패턴

심층 방어 — 한 군데에만 의존하지 않기

모든 보안 사고의 공통점은 "한 줄의 방어가 무너지면 끝"인 구조였다는 것입니다. 다음과 같이 계층마다 독립적인 방어를 두면 한 곳이 뚫려도 전체가 무너지지 않습니다.

01

Segmentation

VLAN/서브넷/보안 그룹으로 폭발 반경을 좁히기. 서버는 서버끼리, 단말은 단말끼리.

02

Authentication & Integrity

802.1X, mTLS, 서명된 응답(DNSSEC), 코드 서명. "출처가 누구냐"를 매번 확인.

03

Confidentiality

TLS, IPsec, 디스크/메시지 암호화. 경로가 빼앗겨도 내용은 무사하게.

04

Observability

플로우 로그, IDS/EDR, 이상 탐지. 모르면 막을 수 없다 — 보이게 하는 것 자체가 방어.

05

Least Privilege

네트워크 ACL, IAM, 토큰 범위 제한. 침해되어도 할 수 있는 일을 줄임.

06

Zero Trust

"내부망이라 안전하다" 가정을 버리기. 매 요청마다 신원·기기·권한을 다시 확인.

체크리스트

실무 점검

  1. 모든 외부 트래픽 HTTPS, HSTS 활성화
  2. 인증서 자동 갱신(ACME), 만료 알람
  3. 리졸버에 DoT/DoH 적용, 가능한 곳에 DNSSEC 검증
  4. 스위치에 DAI/DHCP snooping, 게스트망 분리
  5. 방화벽: 기본 거부, 명시 허용. 출발지 IP 검증(BCP38) 가능 영역에서
  6. L7: WAF + 입력 검증 + 의존성 취약점 스캔
  7. 관측: 흐름 로그, DNS 로그, 인증 시도 로그 보관·경보
  8. 모의훈련: 정기적인 침해사고 대응 훈련(테이블탑·기술훈련)
