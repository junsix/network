계층 모델 — 책임을 나눠 복잡도를 길들이기

네트워크는 거리, 매체, 신뢰성, 의미 등 성격이 다른 문제들이 한꺼번에 묶여 있습니다. 계층 모델은 이 문제들을 독립적으로 풀 수 있게 분리하기 위한 약속입니다. 각 계층은 위 계층에는 "내가 알아서 해주겠다"고 약속하고, 아래 계층에는 "이 정도만 부탁한다"고 위임합니다.

비교

OSI 7계층 vs TCP/IP 4계층

OSI는 교과서적인 분류, TCP/IP는 실제로 인터넷이 동작하는 방식에 가까운 분류입니다.

OSITCP/IP실무 비고
L7 Application
L6 Presentation
L5 Session		Application대부분의 앱 프로토콜이 5~7층을 한꺼번에 책임짐 (예: HTTP는 인코딩/세션/의미를 모두 다룬다).
L4 TransportTransportTCP/UDP/QUIC. "호스트 안의 프로세스"를 가리키는 포트 개념이 여기서 등장.
L3 NetworkInternetIP 주소가 핵심. "어떤 네트워크의 어떤 호스트인가."
L2 Data Link
L1 Physical		Network Access (Link)이더넷·Wi-Fi 등 매체별 규격. 실제 비트가 흐른다.
캡슐화

위에서 내려갈수록 헤더가 붙는다

애플리케이션이 만든 바이트는 아래 계층을 거치며 헤더가 차곡차곡 붙고, 받는 쪽은 위로 올라가며 헤더를 하나씩 벗겨냅니다. 이것이 "캡슐화/디캡슐화"입니다.

왜 이렇게 나눌까? 만약 모든 계층이 한 덩어리라면, Wi-Fi에서 이더넷으로 바뀔 때 HTTP 코드도 같이 고쳐야 합니다. 계층 모델 덕분에 "케이블이 바뀌어도 HTTP는 그대로"인 세상이 가능해진 겁니다.

계층별 보안 고려사항

계층대표 위협대응 위치
L2ARP 스푸핑, MAC flooding, 비인가 AP스위치 보안(DAI, port security), 802.1X
L3IP 스푸핑, 라우팅 하이재킹, ICMP 남용uRPF, BGP RPKI, ACL
L4SYN flood, 포트 스캐닝SYN cookie, 방화벽, rate limit
L7SQLi, XSS, 인증 우회WAF, 입력 검증, 인증/인가 설계
L6/7도청, 위변조TLS (기밀성·무결성·인증)
한 계층의 방어만으론 부족합니다. 예를 들어 TLS(L6/7)는 도청은 막지만, ARP 스푸핑(L2)으로 트래픽이 다른 곳으로 흐르는 것 자체는 막지 못합니다. 그래서 심층 방어(defense in depth)가 필요합니다.
